Gallery Manager

GitHub repo
Live demo

A group project completed for a 3rd year CPSC Database Management course. Worked collaboratively with a small group to design and implement a database using SQL and Oracle, designing ER diagrams, schemas, and normalizing to show our learning process. In brief, our database was created to allow art and gallery owners to keep track of art piece locations, art galleries, and visitors.

For this project, we also designed a front end using PHP, JS, CSS and HTML, allowing users to access the database, track art piece locations, and see who has recently visited an art piece. We also implemented the ability to manage the existing visitors in the database, including adding a new visitor with the required information, updating existing visitor information, and deleting a visitor from the database completely.

SOUUP: A Mental Health Check-In Site

GitHub repo

Souup was the result of a hackathon project implemented by a small team, and completed under a time constraint of 24 hrs. The idea behind this site was to provide a quick and simple way for users to let close ones know that they were ok and safe in the event that they find it difficult to reach out. This way, instead of initiating a conversation, the user simply has to open the site, log in, and press a button to send a pre typed out message.

To implement the simple messaging system, we used a chat API and JS so users would be able to send the message quickly. Additional features of the site included a music playlist embed of instrumental music as white noise, and the ability to customize the background of the site, all designed to help users feel more comfortable while on the site.

Text Adventure Game

GitHub repo

This project is a (currently in need of a better name) project created for a 2nd year CPSC course on Object-Oriented Programming. As per the title, it's a game where the user is able go on an adventure and choose which path to take. Each path leads to an obstacle, where user chooses an action and either gains or loses progress. The main goal is to get to 100% progress, with the user able to save and load their user data in case they aren't able to finish in one sitting. Implemented using Java and Swing, and tested thoroughly with JUnit.
